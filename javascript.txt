1. Question: What is JavaScript?

7. Question: What is the difference between == and ===?
   Workflow: `==` checks for value equality, while `===` checks for both value and type equality.
   Real-Life Example: Comparing a string to a number.
   Hinglish Explanation: `==` sirf value ki equality check karta hai, jabki `===` value aur type dono ki equality check karta hai.
   English Translation: `==` checks for value equality, while `===` checks for both value and type equality.
   Interview Answer: `==` performs type coercion if necessary, whereas `===` does not allow type coercion.

    Interview Answer: `null` is intentional absence of any value, while `undefined` signifies that a variable is declared without a value.

    Interview Answer: JavaScript hoists declarations but not initializations, meaning you can use variables before they are declared.
